# Brain Tumor Detection 🧠

A YOLOv8-based object detection system that identifies brain tumors in MRI images,
classifying regions as **positive** (tumor present) or **negative** (no tumor).

> ⚠️ **Research/learning project only.** Trained on a small 2-class demo dataset.
> Not validated for clinical use — do not use for real diagnostic decisions.

**🔗 Live Demo:** [braintumerdetection-isamadafridi.streamlit.app](https://braintumerdetection-isamadafridi.streamlit.app/)

## Option 1: Streamlit Cloud (Simplest — no Docker, no API)

This approach loads the model directly in Streamlit. No separate API process needed.

### Files needed
- `streamlit_app.py`
- `streamlit_requirements.txt` (rename to `requirements.txt` in your repo)
- `best.pt`

### Steps

**1. Push to a public GitHub repo:**
```bash
git init
git add streamlit_app.py requirements.txt best.pt
git commit -m "initial commit"
git remote add origin https://github.com/your-username/your-repo.git
git push -u origin main
```

**2. Deploy on Streamlit Cloud:**
- Go to [share.streamlit.io](https://share.streamlit.io)
- Click **New app**
- Select your GitHub repo
- Set main file to `streamlit_app.py`
- Click **Deploy**

**3. First deploy takes 2-3 minutes** — Streamlit installs all dependencies fresh.
After that, the model stays cached between requests via `@st.cache_resource`.

### Run locally
```bash
pip install -r requirements.txt
streamlit run streamlit_app.py
```

## Option 2: FastAPI + Streamlit (Local / Docker)

This approach runs two processes in one container:
- **FastAPI** on port `8000` — handles model inference
- **Streamlit** on port `7860` — serves the UI, calls FastAPI internally

### Why two processes?
Separating inference into an API means you can call it independently (curl, Postman,
or any other frontend), not just from Streamlit.

### API Endpoints

**Health check:**
```bash
GET http://localhost:8000/health
# → {"status": "ok"}
```

**Run inference:**
```bash
POST http://localhost:8000/predict?conf=0.25
Content-Type: multipart/form-data
Body: file=<your_image.jpg>

# Response:
{
  "num_detections": 2,
  "detections": [
    {
      "class": "positive",
      "confidence": 0.612,
      "box_xyxy": [120.5, 80.3, 340.1, 290.7]
    }
  ]
}
```

**Interactive API docs** (auto-generated by FastAPI):
```
http://localhost:8000/docs
```

### Run locally (without Docker)

**Terminal 1 — start API:**
```bash
pip install -r requirements.txt
uvicorn api:app --host 0.0.0.0 --port 8000 --reload
```

**Terminal 2 — start UI:**
```bash
streamlit run app.py
```

Open `http://localhost:8501` in your browser.

## Option 3: Docker (Single container, FastAPI + Streamlit)

Everything runs in one container. Streamlit (port 7860) talks to FastAPI (port 8000)
internally — only port 7860 is exposed.

### Prerequisites
- Docker installed
- `best.pt` in the same folder as `Dockerfile`

### Build
```bash
docker build -t brain-tumor-app .
```

> First build takes several minutes — downloads Python, installs torch + ultralytics.
> Image size is ~2-3GB with CPU-only torch (the Dockerfile forces this explicitly
> to avoid pulling the full CUDA build which would push it to ~9GB).

### Run
```bash
docker run -p 7860:7860 brain-tumor-app
```

Open `http://localhost:7860` in your browser.

### How the container starts

`start.sh` is the entrypoint:
```bash
uvicorn api:app --host 0.0.0.0 --port 8000 &   # FastAPI in background
sleep 5                                           # wait for model to load
streamlit run app.py --server.port 7860          # Streamlit in foreground
```

### Deploy Docker container to Hugging Face Spaces

1. Create a new Space on [huggingface.co/spaces](https://huggingface.co/spaces)
   — choose **Docker** as the SDK, port `7860`
2. Clone the Space repo:
```bash
git clone https://huggingface.co/spaces/your-username/your-space-name
```
3. Copy files into it:
```bash
cp api.py app.py Dockerfile start.sh requirements.txt README.md best.pt your-space-name/
```
4. Push:
```bash
cd your-space-name
git add .
git commit -m "deploy"
git push
```
HF builds the Docker image automatically. First build takes ~5 minutes.

## Common Errors

| Error | Cause | Fix |
|---|---|---|
| `ImportError: import cv2` | OpenCV missing | Add `opencv-python-headless` to `requirements.txt` |
| `No detections` at conf=0.25 | Low-confidence image | Lower threshold to 0.10, or try a clearer MRI scan |
| Docker image ~9GB | CUDA torch installed | Dockerfile already forces CPU-only torch via `--index-url https://download.pytorch.org/whl/cpu` |
| Streamlit Cloud cached old deps | requirements.txt updated but not reloaded | Go to Manage App → Reboot App |
| `localhost:8000` connection refused on Streamlit Cloud | FastAPI not running there | Use `streamlit_app.py` (standalone), not `app.py` (which needs FastAPI) |

## Training (how best.pt was produced)

```python
from ultralytics import YOLO
model = YOLO('yolov8s.pt')
model.train(
    data='brain-tumor.yaml',
    epochs=100,
    imgsz=640,
    batch=16,
    workers=8,
    patience=20,
    project='brain_tumor_runs',
    name='exp1'
)
```

Checkpoints saved every epoch to Google Drive to survive Colab disconnects.
Resume after disconnect:
```python
model = YOLO('path/to/last.pt')
model.train(resume=True)
```
